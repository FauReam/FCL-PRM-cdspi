## 一、旧方向为何失败

### 旧命题（CD-SPI 诊断框架）

> CD-SPI 诊断框架区分联邦聚合中的噪声型发散与信号型发散。低容量配置（head-only）产生噪声发散，全参数 FT 产生结构化信号发散。

### 实验证伪

| 配置 | CD-SPI sym | EVR | CKA |
|------|:---:|:---:|:---:|
| head-only GELU (ReLU+ReLU) | 0.0011 | 0.0000 | 1.0000 |
| head-only ReLU | 0.0011 | 0.0000 | 1.0000 |
| full-FT feasibility 2r | 0.0016 | 0.0000 | 1.0000 |
| M2 centralized full-FT | — | — | — |

**所有配置 CD-SPI sym 一致 ≈ 0.001**。诊断工具给所有患者开出同一张化验单——工具与现象之间没有区分度。

### 根因分析

1. **数据过小**（19k 样本 / 1.4B 参数 = 1:74000），训练必然过拟合。过拟合状态下的发散测量没有意义。
2. **CKA=1.0** 说明 Pythia-1.4B pretrained backbone 的表征空间几乎不可变——测量对象本身不动，测量工具自然无效。
3. **CD-SPI 的测量空间选错了**：backbone 倒数第二层 hidden state 的余弦距离捕捉不到联邦训练中真正的发散（梯度结构、输出分布、attention pattern）。
4. **导师的核心质疑未回答**：即使测到了噪声 vs 信号，不去噪有实际危害吗？

## 二、导师谈话精要（2026-06-18）

### 学生的观测

> "冻结 backbone 反而比全参数发散更大"
> "CD-SPI 差距极其小，结果是阴性的"

### 导师的回应（按逻辑链）

1. **"如果是噪声你怎么办？丢掉？"**
   → 必须证明去噪有实际收益，否则研究噪声就没有操作性定义。

2. **"噪声的原因是什么？"**
   → 必须有理论解释为什么冻结 backbone 产生噪声型发散。

3. **"为什么冻结 backbone 必然产生噪声型发散？"**
   → 如果无法证明必然性，就只是一个现象描述而非科学贡献。

4. **"噪声在不同水平会有什么影响？"**
   → 需要量化噪声-性能的因果关系，而非二元分类。

5. **"如果不去掉噪声，对学习效果的影响是什么？"**
   → **核心判断**：必须先证明去噪有用，再研究噪声。如果去噪不影响性能，整个研究问题就不成立。

6. **"联邦学习有些方法是天然去噪声的，这个前提要验证。"**
   → FedAvg 本身的聚合就是去噪——需要证明残余噪声仍然构成问题。

### 导师暗示的路径

- 从数学上刻画冻结 backbone → 子空间约束 → 噪声主导发散 → 聚合失效的因果关系
- 必须先做"去噪 vs 不去噪"的性能对比实验
- 理论要能给出条件：在什么参数容量下，发散从信号退化为噪声

| 论文 | 关键结论 | 与本方向的关系 |
|------|---------|:---:|
| **FedAPM** (KDD 2025) | 理论上证明 partial model personalization 会**加剧** client drift | 直接支持 |

| **Momentum+SCAFFOLD** (ICLR 2024) | 动量消除异构性假设 | 理论工具 |

### 空白

这些工作都关注 "限制参数 → drift 增大" 这一事实，但**没有人从发散性质（信号 vs 噪声）角度给出理论解释**。这个空白恰好是你的切入点。

## 四、可行的新方向（选项，非决定）

### 方向 A：容量-发散性质-聚合质量因果理论

```
核心命题：
联邦学习中参数容量限制不仅影响发散量，更根本性地改变
发散的性质——存在容量阈值，低于该阈值发散从信号主导
（结构化、可聚合）退化为噪声主导（随机、聚合失效）。

三个 Lemma：
  L1: 子空间约束下客户端发散方差可分解为信号项 + 噪声项
  L2: 噪声项比例与 head 维度成反比 → 存在容量阈值
  L3: 噪声主导的发散导致 FedAvg 泛化误差上界增大

实验：
  容量连续谱（head / LoRA r=8/64/256 / full）× EVR 趋势
  去噪干预实验（去噪后低容量应提升，高容量不变）
  正控制实验（合成已知发散验证灵敏度）
```

**优点**：有理论深度，不依赖 full-FT 优于 head-only 的性能主张。
**风险**：去噪实验如果显示噪声不影响性能，整个框架崩塌。

### 方向 B：Null Result — 容量不影响联邦 PRM 发散结构

```
核心命题：
系统性地证明：在 Pythia 系列 + FedAvg 基线下，
客户端参数容量对表征发散的结构性质无影响。
CD-SPI sym 在所有配置下一致 ≈ 0.001。

这比阳性结果更有价值——因为直觉预期是容量有影响。

需要：
  TOST 等价性检验（统计证明"无差异"）
  正控制实验（证明 CD-SPI 灵敏度本身没问题）
  三方交叉验证（CD-SPI + CKA + JS 一致）
```

**优点**：已有数据就可以写。审稿人喜欢反直觉的 null result。
**风险**：审稿人可能认为"没有发现"不够分量。

### 方向 C：转向纯理论

```
不做新实验。在现有数据基础上，从理论上证明：
为什么子空间约束在 FL 中必然导致噪声型发散。

需要的工具：
  - 高维概率（随机矩阵理论）
  - 联邦优化（SCAFFOLD 收敛界扩展）
  - 表征学习（维度坍缩的微分方程刻画）
```

**优点**：不依赖更多实验数据。
**风险**：纯理论论文在 ML 会议发表难度大。
