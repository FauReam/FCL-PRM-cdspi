H1 — 低容量 = 噪声发散
当只有 PRM head (256-dim MLP) 参与训练、backbone 冻结时：
· CD-SPI (symmetrical) ≈ 0
· PCA EVR ≈ 0
· CKA ≈ 1.0
· 跨客户端差异不可与随机噪声区分
→ FedAvg 聚合无信息增量
H2 — 高容量 = 结构化发散
当更多参数参与训练 (LoRA r=256 / full FT) 时：
· CD-SPI (symmetrical) > 0.01
· PCA EVR > 0.3
· CKA < 1.0
· 跨客户端差异呈现统计显著的结构
→ FedAvg 聚合产生有意义的差异化更新

为什么选 LoRA r=256 而不是 full FT？
LoRA r=256 是 batch=128 组里容量最高的配置，12.5h 出 5 轮数据，符合"单 config 一天内验证"原则。 full FT (batch=4) 跑 5 轮需要 ~6 天，留给 Phase 0 主阶段。
如果 LoRA r=256 已经能看到 CD-SPI 信号，那 full FT 只会更强 —— 验证通过。

💡 关键解读
CD-SPI (symmetrical backbone) = 0.0011 意味着：从 frozen Pythia-1.4B 倒数第二层提取的 hidden state，在 4 个客户端之间几乎完全一致。 无论 client 是 math / code / medical / general，模型对同一个推理步骤的表示是相同的。

CD-SPI (head embedding) = 0.0867 是假信号：原始 CD-SPI 测量的是 PRM head 中间层特征，包含了 4 个 client 各自随机初始化的 head 权重噪声。 这正是 Symmetrical CD-SPI 设计要解决的问题 —— 必须在 backbone 统一层面对齐测量空间。

PCA EVR = 0.0 + CKA = 1.0 双重确认：跨客户端嵌入矩阵无结构差异（PCA 第一主成分不解释任何方差），特征空间完全对齐（CKA 满分）。 三个独立指标同时指向同一结论：head-only training produces zero structured cross-client divergence.
